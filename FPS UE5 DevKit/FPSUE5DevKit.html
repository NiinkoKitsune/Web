 Purpose :                                                                                          

The purpose of this user manual is to provide technical support to install and use Hellkit in order to create Doom like game using Unreal Engine 5.0.3

  • The possibility of creating custom weapons, passing from the global statistics of the weapon to the projectiles fired.

  • Create custom projectiles, change its trajectory, its bullet drop, its behavior.

  • Create enemies with different statistics, modifiable behavior, whether allies or enemies, a simple button is enough to change that. The player will have the possibility of embodying his enemy or ally to take advantage of the skills specific to the entity.

  • Easily editable maps with fully configurable spawners to create the number of monsters you want with the interval you want.

Prerequisite :                                                                                

  1. Game Engine

HellKit 0.0.1 requires installing Unreal Engine 5.0.3 to run properly. You can find the game engine at this address: Download Unreal Engine .

Create an Epic Games account and open the launcher. On top left of the

launcher click on Unreal Engine.

On top click on “Library” then click on the “+” on right of Engine Version (Under Library)

                       

A new box will appear, select the version by clicking on version number (A) and choose “5.0.3”, then click on install (B).

  1. Dev Kit

Next step is to install HellKit 0.0.1 in Unreal Engine. The dev kit is available at this address:  FPS-UE5-DevKit - Github 

Instructions :

  • Click on “Code” (A) then download the .zip (B)                                         →

  • Unzip the downloaded file.

IMPORTANT Before launching Unreal Engine, Go in Plugins folder and then Unzip TAPython_5_0_3_win64.zip

If you miss this step, you will have multiple errors with the editor.

  • You can now launch the unreal you downloaded before (When installed, “Install” button will be changed to “Launch”)

  • You will not see the project, it’s normal you will have to click on browse on the bottom left of new window

  • Then find your Unzipped fill of the project and double click on “DevKitDoomLike.uproject”.

The project is now ready and will be launched.

GLOBAL TOOL                                                                        

You have multiple tools in devkit, you’ll probably see the same part in multiple tools, here is the explanation.

CREATE NEW ASSET:

        With tool you can easily create new assets, based on which tool you used (Projectiles tool will create projectiles assets, etc)

Name : The name of new asset

File Path : The path of the new asset (Game first is the main folder of the Game (also known as “Content”))

Unique Name? : If an asset with the same name already exists, it will add a number after the name (BP_NewProjectile will be BP_NewProjectile1 etc…) otherwise the asset won’t be created.

SELECT ASSETS :

You can easily select your new asset by clicking on None and select the name of the created one.

!! IMPORTANT !!

NEVER SELECT BP_BaseAssets (for example BP_BaseProjectile, BP_BaseWeapon) THEY ARE THE ROOT OF EVERYTHING AND CAN RESULT TO BUG IN GAME.

DELETE ASSETS :

Don’t manually delete assets created with Tool, use the delete button (be careful, action is undoable).

TIPS :

There is multiple bar like this in the tool :

This is a slider, with min and max value. If you want to set a value below or above, click on number and set it manually. Slider max and min is just to give you an idea of the range you should select.

INTERACTIVE OBJECT :

You can create interactive objects in level, like buttons and doors which every Entity (Player and Possessed Creature) can interact with.

EVERY INTERACTIVE OBJECT HAVE THE FOLLOWING BEHAVIOR (Until   ▶BUTTON )

Interactive (BP_BaseInteractive):

        By clicking on None(A), you can select item on scene through a list(only Interactive one)

You can set the focus on the object in scene with (B)

You can use pipette (C) if you want to select an object directly on scene.

Show Interact UI (bool):

        Does the UI appear when the player is near it?

Delay Before Deactivate? (bool):

        Should interactive item self-deactivate after X time?

Delay Before Deactivate (float):

        Time before the interactive item self deactivates.

Unlock Interact (array BP_BaseInteractive):

Represented by Red line (See Attachment):

If not empty, other interactive objects in the array need to be activated before activating the one currently modified.

Auto Interact (Struct S_AutoInteractObject):

        Represented by Green line (See Attachment):

Auto interact will automatically interact with an object, there is 2 element in struct :

  • Interact Object (BP_BaseInteractive) : The other interactive object which will be Auto Interact.
  • Delay before Activation (float) : In second, the time before the Auto Interact

▶        BUTTON :

A simple button to interact with, No more behavior.

▶        DOOR:

On activation the static mesh “Door” will be set to invisible and will not block the player anymore.

On desactivation the static mesh “Door” will be set to visible and block the player.

▶        CHECKPOINT :

Player can interact with the checkpoint, and when he dies, he will respawn on it.

The checkpoint is only for the main character, mean that an entity that interact with won’t respawn on it if he died.

▶        SPAWNER :

Spawner have few more behavior :

  • Monster To Spawn (BP_ControllableEntity class): The monster which will spawn.
  • Max Monster to Spawn (int) : The max amount of entities to spawn with the current spawner.
  • Delay Between Each Spawn (float): In seconds, time between each spawn.
  • Behavior Tree (behaviorTree) : The behavior of the spawned A.I., See the section about A.I.
  • Kill On End Possesse? (bool) : Does the entity die when possession is finished?

Interacting with spawner will disable it, mean that no more entity will spawn.
